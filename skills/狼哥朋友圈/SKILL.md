# 狼哥朋友圈 — 每日内容规划

你帮狼格拉底做每日朋友圈内容的完整规划：从扫素材到写文案到匹配配图，最后输出一份可直接执行的规划文档。

核心框架文件在 `03-意心会/03-内容/狼格拉底IP/朋友圈/朋友圈内容框架.md`，那是操作手册，本 Skill 的所有规则以那个文件为准。**执行前必须先读取该文件。**

---

## 执行流程

### Step 1：读框架

读取 `03-意心会/03-内容/狼格拉底IP/朋友圈/朋友圈内容框架.md`，加载：
- 常量（在售产品、关键词、核心观点）
- 6种内容类型定义
- 每日结构模板
- 写作手法清单
- 素材来源索引

### Step 2：扫素材

按框架中"素材来源索引"列出的路径，扫描以下位置的**近期内容**：

| 扫描位置 | 找什么 |
|---------|-------|
| `05-归档/01-会话日志/` | 最近2-3天的会话日志，提取可转化的事件、洞察、AI实操 |
| `01-知识体系/07-录音分析/` | 近期录音萃取的观点、金句 |
| `03-意心会/04-运营/活动/` | 近期直播/活动的复盘文档 |
| `03-意心会/02-客户/` | 客户动态、成果 |
| `01-知识体系/02-知识单元/` | 可转化的观点 |
| `01-知识体系/04-金句库/` | 可直接用的金句 |

不需要全部扫完——按"最近发生的事优先"原则，挑出8-10个可用素材点就够了。

如果用户在对话中已经提到了今天的素材（比如"今天跟XX聊了""刚做了个XX"），这些是最高优先级的素材，直接用。

### Step 3：列素材池 + 匹配类型

把扫到的素材列成表：

```
| 素材 | 来源 | 适合类型 | 命中关键词 |
```

每个素材标注最适合的内容类型（A~F）和能命中的关键词。

### Step 4：规划当日内容

从素材池中挑4-5条，按框架中的每日结构模板排列：

**标准日**：
```
[早/午] 1条轻的 — F类
[晚间] 2-3条重的 — A/B/C类交替，不重复类型
[临睡] 1条轻的 — F类收尾
```

**开课日/活动日**和**空白日**用框架中对应的模板。

### Step 5：写文案

为每条写出可发布的文案。

**轻内容（F类）**：
- 不超过3句话
- 不解释不铺垫，直接说
- 可以是一个问题、一句感悟、一个观察

**重内容（A/B/C类）**：
- 必须有具体案例、数据或故事支撑——框架是骨头，案例是肉
- 结构：切入（别人的问题/数据/反差）→ 拆解（2-3个要点）→ 金句或核心观点收尾
- 引流语永远在最后一句，且不超过全文20%

**写作铁律**（从框架文件加载，这里列核心几条）：
- 不说"我觉得"，说"闭门会上被问最多的问题是"/"课堂数据显示"
- 数据替代形容词："38单+320到店"比"效果好"有力100倍
- 核心观点用不同案例反复锤，不是复制粘贴同一句话
- 绝不以引流条收尾

### Step 6：匹配配图

为每条内容匹配配图方案：

**配图来源优先级**：

1. **手机近照** — 如果内容涉及日常、场景、现场，调 `/相册管理` skill 扫描近期手机照片，看有没有合适的
2. **意心会素材库** — 扫 `03-意心会/07-素材/` 下的对应目录：
   - 课程/活动相关 → `活动记录/`
   - 客户相关 → `客户见证/`
   - 产品相关 → `产品/`
   - 人物/合照 → `人物/`
   - 工作过程 → `工作过程/`
3. **无合适配图** — 标注建议：
   - "建议拍照：XX场景" — 需要真实照片的
   - "建议制图：用 /极简抽象配图" — 适合配概念图的
   - "纯文字即可" — 有些F类随感/提问不需要图

每条内容旁标注配图方案，格式：`📷 配图：[来源] 描述`

### Step 7：检查清单

跑一遍检查：

- [ ] 轻→重→轻节奏
- [ ] 不以引流收尾
- [ ] 每条类型不重复
- [ ] 至少1条非商业
- [ ] 核心观点至少出现1次
- [ ] 4个关键词至少覆盖3个
- [ ] 重内容每条都有案例/数据
- [ ] 引流条≤总量1/4

### Step 8：输出

创建 `03-意心会/03-内容/狼格拉底IP/朋友圈/YYMMDD/规划.md`，包含：
- 素材池表格
- 每条内容（含文案全文 + 类型标注 + 关键词命中 + 配图方案）
- 检查清单

## 用户说"这个可以发朋友圈"时

不需要走完整流程。直接把用户提到的素材：
1. 判断属于哪个类型（A~F）
2. 按写作规则写出1-2个版本
3. 建议配图
4. 追加到当天的规划文件（如果已有）或告诉用户

---

## 语言

- 文案用口语化中文，不书面语
- 遵循《中文文案排版指北》
- emoji 适度（1-3个点缀），不堆砌
